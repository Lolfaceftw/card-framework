### 1. Is the TREC 2020 Dataset a good choice?

**Yes, it is excellent, but with one critical caveat.**

- **The Good:** It provides a standardized "Ground Truth." In research, you cannot just say "it looks good to me." You need to compare against human baselines. The NIST scores (E/G/F/B) and binary attributes give you a target to beat.
- **The Caveat (The "Format Mismatch"):**
  - **TREC Summaries:** These are typically _descriptions_ written for a podcast app (e.g., "In this episode, Joe talks about aliens...").
  - **Your Pipeline:** Your pipeline generates a _script_ for a Voice Cloner (e.g., "Speaker A: Welcome back. Today we discuss aliens...").
  - **The Risk:** If you calculate ROUGE/BERTScore between a _Description_ (Reference) and a _Script_ (Candidate), the scores will be artificially low because the formats differ, even if the content is correct.
  - **The Fix:** You must instruct your LLM (Stage 2) to output a **summary description** for the sake of the benchmark, OR you must accept that ROUGE scores will be lower and rely more heavily on the **LLM-as-a-Judge** metrics (which can understand that a script containing the same facts as the description is still "correct").

### 2. Are your metrics research-grade?

Your proposed list is very close to the current State-of-the-Art (SOTA). To make it truly "Research Grade" for 2025/2026, you should structure them into three tiers.

**Tier A: Reference-Based (Standard NLP)**

- **ROUGE-L:** Measures structural similarity (longest common subsequence). Good for catching exact phrasing.
- **BERTScore (F1):** Measures semantic similarity using embeddings. This is crucial because it handles synonyms (e.g., "The host discussed" vs "The speaker talked about") better than ROUGE.

**Tier B: Reference-Free (LLM-as-a-Judge / G-Eval)**

- _Note: "Fluency" is usually unnecessary for modern LLMs (GPT-4/DeepSeek rarely make grammar errors). Focus on these three:_
- **Faithfulness (Hallucination Check):** Is the summary supported by the transcript? (Critical).
- **Coherence (Flow):** Does the summary make logical sense?
- **Relevance (Informativeness):** Does it capture the _main_ points compared to the ground truth?

**Tier C: Pipeline-Specific (The "Engineering" Metrics)**

- **Duration Adherence:** (Specific to your project) Does the output fit the time constraint?
- **Speaker Consistency:** (Specific to your project) Did the LLM invent speakers not in the input?

class PodcastBenchmark:
    def __init__(self):
        print("Loading metrics...")
        self.rouge = load("rouge")
        self.bertscore = load("bertscore")

    def evaluate_g_eval(self, transcript: str, summary: str) -> Dict[str, float]:
        """
        LLM-as-a-Judge (G-Eval) to score Faithfulness, Coherence, and Relevance.
        Returns scores 1-5.
        """
        # Truncate transcript to fit context window if necessary
        trunc_transcript = transcript[:15000]

        prompt = f"""
        You are an expert evaluator for podcast summarization.

        Source Transcript:
        {trunc_transcript}...

        Generated Summary:
        {summary}

        Evaluate the Generated Summary on the following criteria (Score 1-5):

        1. Faithfulness: Is the information in the summary factually supported by the transcript? (1=Hallucinated, 5=Fully supported)
        2. Coherence: Is the summary well-structured and logical? (1=Incoherent, 5=Perfect flow)
        3. Relevance: Does the summary capture the most important information? (1=Irrelevant, 5=Captures all key points)

        Return ONLY a JSON object: {{"faithfulness": int, "coherence": int, "relevance": int}}
        """

        try:
            response = CLIENT.chat.completions.create(
                model=MODEL_NAME,
                messages=[{"role": "user", "content": prompt}],
                temperature=0,
                response_format={"type": "json_object"}
            )
            return json.loads(response.choices[0].message.content)
        except Exception as e:
            print(f"G-Eval Error: {e}")
            return {"faithfulness": 0, "coherence": 0, "relevance": 0}

    def run_benchmark(self, dataset_path: str, output_csv: str, limit: int = None):
        """
        dataset_path: Path to the TREC json file (list of dicts)
        """
        with open(dataset_path, 'r') as f:
            data = json.load(f)

        if limit:
            data = data[:limit]

        results = []

        print(f"Starting benchmark on {len(data)} episodes...")

        for entry in tqdm(data):
            episode_id = entry.get('episode_id', 'unknown')
            transcript = entry.get('transcript', '')
            human_summary = entry.get('summary', '') # The Ground Truth

            if not transcript:
                continue

            # 1. Generate Candidate Summary (Your System)
            candidate_summary = self.generate_summary(transcript)

            # 2. Compute Reference-Based Metrics (ROUGE & BERTScore)
            rouge_scores = self.rouge.compute(
                predictions=[candidate_summary],
                references=[human_summary]
            )

            bert_scores = self.bertscore.compute(
                predictions=[candidate_summary],
                references=[human_summary],
                lang="en",
                device=self.device
            )

            # 3. Compute Reference-Free Metrics (G-Eval)
            g_eval_scores = self.evaluate_g_eval(transcript, candidate_summary)

            # 4. Compute Pipeline Constraints (Length)
            # Assuming standard podcast speaking rate ~150wpm
            word_count = len(candidate_summary.split())
            est_duration_min = word_count / 150.0

            # Store Results
            row = {
                "episode_id": episode_id,
                "rouge1": rouge_scores['rouge1'],
                "rougeL": rouge_scores['rougeL'],
                "bertscore_f1": bert_scores['f1'][0],
                "faithfulness": g_eval_scores['faithfulness'],
                "coherence": g_eval_scores['coherence'],
                "relevance": g_eval_scores['relevance'],
                "word_count": word_count,
                "est_duration_min": est_duration_min,
                "generated_text": candidate_summary,
                "ground_truth_text": human_summary
            }
            results.append(row)

        # Save to CSV
        df = pd.DataFrame(results)
        df.to_csv(output_csv, index=False)

        print("\n--- Benchmark Results ---")
        print(f"Avg ROUGE-L: {df['rougeL'].mean():.4f}")
        print(f"Avg BERTScore F1: {df['bertscore_f1'].mean():.4f}")
        print(f"Avg Faithfulness (1-5): {df['faithfulness'].mean():.2f}")
        print(f"Avg Relevance (1-5): {df['relevance'].mean():.2f}")
        print(f"Results saved to {output_csv}")

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--dataset", required=True, help="Path to TREC json dataset")
    parser.add_argument("--output", default="benchmark_results.csv")
    parser.add_argument("--limit", type=int, default=None, help="Limit number of episodes for testing")
    args = parser.parse_args()

    bench = PodcastBenchmark()
    bench.run_benchmark(args.dataset, args.output, args.limit)

### 4. How to interpret the results for your paper/study

1.  **BERTScore > ROUGE:** Since your output format (script) differs from the ground truth (description), expect ROUGE to be lower (maybe 0.2 - 0.3). **BERTScore** is more important here; it should be higher (0.85+) because it measures if the _meaning_ is similar, even if the words are different.
2.  **Faithfulness is King:** For a podcast summarizer, a Faithfulness score < 4.0 is a failure. It means the model is hallucinating content.
3.  **Relevance vs. Duration:** Plot "Relevance" against "Duration." A good system maintains high relevance even as you force the duration down (e.g., 2 minutes).
